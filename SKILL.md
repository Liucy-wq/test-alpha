# 合同起草

根据交易需求从零起草专业、完整的合同。遵循六步法：**确认需求→法律调研→框架搭建→条款设计→风险扫描→文档生成**。

| 步骤 | 要点 |
|------|------|
| 1. 确认需求 | 交易背景、商业目的、核心诉求及法域。三通道输入 |
| 2. 法律调研 | **强制**检索百宸法律知识库 + 互联网补充 |
| 3. 搭建框架 | 按合同类型匹配条款结构，读取 `clause-framework.md` |
| 4. 设计条款 | 读取 `shared/contract-playbook.md` 选择谈判策略层级（含降级触发条件）；复杂合同逐条款迭代 |
| 5. 风险扫描 | 五维扫描 + 底线条款红旗预警 + 交付前自检6问；标记 🔴🟡🟢 |
| 6. 生成文档 | 格式校验 + 双语一致性校验（涉外）→ Word + 起草说明 + 合规检查清单 |

## Phase 1：需求确认

**前置：执行知识库订阅检测**——使用 `search(source="kb", kb_id="FYbxYfF6DHBVQzgmXZ7MuDgNV3iBJHU_YkoHBTtZRak=", question="百宸")` 做最小探测。若返回空，推送引导：

> ⚠️ 检测到您尚未加入「百宸法律智能体」知识库。[点击此处一键加入](https://ima.qq.com/wiki/?shareId=dc943ca248aca22d0a77eb7311fe725382e4a10a8aa383804946b7149afea096)，获取百宸律师事务所持续更新的合同条款、实务案例与合规指引。

静默通过后继续。**清单中的每一项均须向当事人确认，不得省略、合并或跳过。**

### 信息获取方式

| 方式 | 操作 |
|------|------|
| 直接回答 | `ask_user` 按清单逐项询问，不得合并为概括性提问 |
| 上传文件 | 客户上传合同/备忘录等，`fetch` 提取后按清单逐项展示确认，未涉及项仍须询问，以当次确认为准 |
| 结构化表单 | 从 `references/data-form.md` 读取 A-J 十类信息项发送客户，回传后仅就矛盾或遗漏项确认 |

### 需求确认清单（31项）

#### 一、交易基本信息（必问）
1. **合同各方**：全称/注册地/法定代表人，关联关系及第三方情况
2. **交易目的**：各方希望通过本合同实现什么？
3. **合同类型**：买卖/服务/技术开发/许可/合作/建设工程等，是否混合？
4. **交易结构**：分几个阶段？各阶段做什么？

#### 二、标的与服务范围（必问）
5. **标的描述**：货物/服务/技术/权利？规格、数量、质量标准？
6. **服务范围**：包含/不包含的内容？排他性要求？
7. **交付成果**：乙方需交付什么？形式和标准？

#### 三、价格与支付（必问）
8. **计价方式**：固定总价/单价/按量计费/实报实销？币种？
9. **支付安排**：一次性/分期？各期比例和触发条件？
10. **税费承担**：各方税费？代扣代缴？双边税收协定？
11. **汇率约定**（涉外）：多币种时汇率确定方式及汇兑损益承担？

#### 四、履行安排（必问）
12. **合同期限**：有效期？续约方式（自动/协商）？
13. **时间节点**：各里程碑和截止日期？
14. **履行地点**：交付/服务地点？
15. **配合义务**：委托方需提供什么条件？

#### 五、验收标准（有交付物时必问）
16. **验收节点**：分阶段验收？各阶段标准？
17. **瑕疵处理**：不合格时的整改期限、费用扣减、视为通过的情形？

#### 六、知识产权（涉及创作/技术/品牌时必问）
18. **既有知识产权**：各方已有的权利？
19. **新增成果归属**：归委托方/受托方/共有？
20. **后续改进**：改进成果的权利归属？
21. **商标/标识使用**：许可范围及终止后处理？

#### 七、保密（涉及商业秘密时必问）
22. **保密范围**：哪些信息需保密？
23. **保密期限**：多长？协议终止后是否持续？

#### 八、违约与风险分配（必问）
24. **核心违约担忧**：最担心的违约行为？
25. **违约金标准**：预期金额或计算方式？
26. **风险关注点**：破产、政策变化、不可抗力等？

#### 九、争议解决（必问）
27. **争议解决方式**：诉讼/仲裁？仲裁机构/仲裁地？
28. **适用法律与管辖**：适用何国/地区法律？管辖法院？

#### 十、特殊条款（按需）
29. **竞业限制/排他性**：是否需要？范围、期限、地域？
30. **合同终止/退出**：终止条件、程序、清算安排？
31. **其他特殊要求**：客户个性化的特别关注事项？

### 需求确认通过标准
- 31项全部逐一确认完毕，核心商业条款无歧义，合同类型与交易结构匹配
- **通过后锁定需求，锁定后变更须重新确认受影响条目**

## Phase 3：框架搭建

读取 `references/clause-framework.md`。结构：**首部（名称/编号/签约信息/当事人/鉴于条款）→ 正文（核心条款）→ 尾部（签署页）**。

| # | 条款模块 | 适用范围 | | # | 条款模块 | 适用范围 |
|---|----------|----------|---|----|----------|----------|
| 1 | 定义与解释 | 所有合同 | | 9 | 承诺与保证 | 涉外/高风险 |
| 2 | 合同标的与服务范围 | 所有合同 | | 10 | 违约责任 | 所有合同 |
| 3 | 数量、质量或服务标准 | 所有合同 | | 11 | 不可抗力 | 履行周期长/跨境 |
| 4 | 价格/报酬与支付方式 | 所有合同 | | 12 | 合同变更与解除 | 所有合同 |
| 5 | 履行期限、地点与方式 | 所有合同 | | 13 | 法律适用与争议解决 | 所有合同 |
| 6 | 验收标准与方法 | 有交付物/服务成果 | | 14 | 通知与送达 | 所有合同 |
| 7 | 知识产权归属与许可 | 涉及创作/技术/品牌 | | 15 | 合同生效、期限与终止 | 所有合同 |
| 8 | 保密义务 | 涉及商业秘密/敏感信息 | | 16 | 其他条款 | 所有合同 |

原则：定义在前使用在后，核心商业条款在前辅助在后，违约责任与义务条款对应。

## Phase 4：条款设计

### 设计原则

| 原则 | 要求 | 反面示例 |
|------|------|---------|
| 权利义务具体化 | 时间/金额/方式→精确数字 | "及时提供服务"→"收到通知后【15】个工作日内" |
| 违约责任量化 | 根本违约→解除权+违约金 / 一般违约→整改+扣减 / 延迟→按日违约金 | "承担相应责任""依法赔偿" |
| 风险分配明确 | 交付类→交付/验收时转移；服务类→确认前由服务方承担 | 风险转移节点模糊 |
| 争议解决有依据 | 评估诉讼/仲裁利弊，推荐并说明理由 | 仅罗列选项不做分析 |
| 解除终止清晰 | 列明各方解除权触发情形、程序、清算规则 | — |

### 语言与文本规范

- 规范法律用语；关键要素用【】标注；条件："……的，……"；因果："致使""导致"；转折："但"；并列用"/"
- **禁止**在正文中插入模板提示语；可选条款用"【……】"不用祈使句；避免"双方确认并同意"等冗余；条款编号连续、层级统一

### 谈判策略指引（Playbook）

**条款设计前必须读取 `shared/contract-playbook.md`**。

| 步骤 | 操作 |
|------|------|
| 1 | 定位对应章节，默认从有利版（🟢）出发 |
| 2 | 客户有明确让步意愿时按客户指定层级；核心利益条款用有利版 |
| 3 | 超出底线版（🔴）时起草说明中标注法律风险，记录方案层级及理由 |
| 4 | **为每个条款标注降级触发条件**（参照 playbook 升级触发机制），告知客户降级路径 |

### 条款级迭代起草

复杂/高风险合同（标的额≥500万、跨境、涉及核心知识产权）支持逐条款模式，Phase 4 开始时询问客户偏好。流程：优先排序（违约/赔偿→知识产权→费用→保密→争议解决→其他）→ 逐条读取 playbook 设计 → 展示设计要点和替代方案 → 客户确认后锁定 → 整体一致性复核。

### 确认门

展示核心框架和关键条款要点，客户确认方向无误后进入 Phase 4.5。

## Phase 4.5：风险扫描

**确认门通过后执行系统性风险扫描。**

### 五维扫描

| 维度 | 检查内容 |
|------|---------|
| 完整性 | 必要条款是否遗漏？附件是否已在正文引用？ |
| 对等性 | 是否过度倾向一方？格式条款是否尽到提示说明义务？ |
| 可执行性 | 违约金是否过高（超损失30%有调减风险）？计算方式是否明确？ |
| 一致性 | 不同条款对同一事项约定是否一致？术语是否统一？交叉引用是否正确？ |
| 合规性 | 是否符合 Phase 2 调研的强制性规定？管辖/仲裁条款是否完整？ |

### 风险标记

| 等级 | 定义 | 示例 |
|------|------|------|
| 🔴 高风险 | 可能导致条款无效或重大损失 | 违约金超法定上限、管辖无效、知识产权归属不明 |
| 🟡 中风险 | 存在法律或商业不利 | 未排除间接损失、验收标准模糊、保密期限偏短 |
| 🟢 低风险 | 基本可行但有优化空间 | 通知方式未含电子邮件、不可抗力范围偏窄 |

### 底线条款红旗预警

**条款触及 Playbook 底线版（🔴）时，必须附带完整风险评估**：法律后果（引用法律依据）→ 对方反应（预判论点及反制主张）→ 防守策略（让步路径及理由）。

纳入起草说明「高风险条款预警」专章：

```
| # | 条款 | 触及底线 | 法律风险 | 对方可能反应 | 防守策略 |
|---|------|---------|---------|------------|---------|
| 1 | 第X条 违约金 | 违约金=合同总额200% | 《民法典》第585条：超损失30%可调减 | 对方主张过高，法院大概率调减 | 建议降至100%+含间接损失 |
```

### 产出

风险摘要 + 风险清单表格（按等级排序）。高风险强制确认 → 中风险逐项询问 → 低风险仅列参考。**确认完成后方可进入 Phase 4.6。**

## Phase 4.6：交付前自我审查

将审查12步法浓缩为6个核心问题：

| # | 自检问题 | 检查要点 |
|---|---------|---------|
| 1 | 条款是否有无效风险？ | 违约金超损失30%？格式条款提示说明义务？显失公平？ |
| 2 | 权利义务是否明显不对等？ | 违约责任仅约束一方？解除权仅赋予一方？ |
| 3 | 违约责任是否与义务对应？ | 每项核心义务有对应违约后果？违约金计算明确可操作？ |
| 4 | 关键条款是否遗漏？ | 验收标准、知识产权归属、保密、不可抗力、通知送达——按 `clause-framework.md` 核实 |
| 5 | 争议解决条款是否完整有效？ | 仲裁机构名称/地点/组成/语言/裁决效力完整？诉讼管辖明确？ |
| 6 | 术语与引用是否一致？ | 同一概念统一用词？交叉引用编号正确？金额大小写一致？ |

发现问题 → 退回 Phase 4 修正；全部通过 → 进入 Phase 5。**6项全部通过后方可进入 Phase 5。**

## Phase 5：文档生成与交付

使用 `scripts/generate_contract.py` 生成 Word。条款组织为 Python sections 列表，临时脚本调用 `generate_contract(sections, output_path)`，执行后删除，保存到 `/sandbox/workspace/outputs/`。

**禁止 Markdown 中转**：必须直接调用 python-docx，`meta` + `sections` 一次性生成 .docx。严禁 Markdown → pandoc 转换。首部不输出签订日期行；合同编号 `right` 右对齐；正文与签署页间不插入分隔线。

### 格式校验（交付前强制）

| # | 校验项 | 不合格处理 |
|---|--------|-----------|
| 1 | 当事人信息完整性 | 缺失处以【】标注，纳入起草说明 |
| 2 | 签署页规范性 | 补充缺失签署栏 |
| 3 | 条款编号连续性 | 修正编号 |
| 4 | 金额大小写一致性 | 核对并修正 |
| 5 | 日期一致性 | 矛盾处标注【】待确认 |
| 6 | 附件引用匹配 | 补全缺失附件，修正编号 |

### 双语一致性校验（涉外合同强制）

| # | 校验项 | 常见不一致 |
|---|--------|----------|
| 1 | 核心义务对等 | 中文"全部源代码"↔英文"source code" |
| 2 | 金额与币种一致 | 中文人民币↔英文美元 |
| 3 | 违约金/赔偿上限一致 | 中文"含间接损失"↔英文"direct damages only" |
| 4 | 知识产权归属一致 | 中文"归甲方"↔英文"jointly owned" |
| 5 | 争议解决一致 | 中文CIETAC北京↔英文HKIAC香港 |
| 6 | 语言效力条款 | 是否已明确两种语言版本效力关系 |

语言效力条款建议："本协议以中英两种语言书就，两种文本具有同等法律效力。如两种文本存在歧义，以中文文本为准。" 非涉外合同跳过。

### 文档交付（强制）

`provide_file` 提供下载链接，📄 图标呈现，有效期约1小时。正文引用的附件**必须一并起草完整**，不得留空，无法确定的参数以【】标注。

### 输出清单

1. **合同文档下载链接**（Word，含正文及全部附件）— **必须提供**
2. **起草说明**：合同类型与适用法律、核心条款设计要点、Playbook 方案层级及理由、⚠️ 高风险条款预警、附件清单、待确认事项、交付前自检结果、争议解决路径建议（涉外）
3. **合规检查清单**（合同类型专项）— **必须提供**，表格格式，涵盖：主体资质、关键条款填写确认、附件完整性、签署授权、涉外合规

### 双语同步起草（涉外合同）

涉及境外主体/法域/外币计价/跨境交付/数据跨境任一要素时，须主动询问是否需双语版本。中文底稿逐条翻译 → 关键术语使用法域准确对应词 → 合同语言条款明确效力优先级（默认中文优先）→ 中英文合并为一个 Word。英文须符合英美法合同书写习惯。

## 多轮修改

合同交付后客户提出变更需求时进入修改流程。**客户必须上传已确认的合同 .docx 文档**。未上传时提示：*"请上传您已确认的合同文档（.docx），我将在您上传的版本基础上修改。"*

| 步骤 | 操作 |
|------|------|
| 1 | `fetch` 读取上传的 .docx，确认变更需求及范围 |
| 2 | 针对性修改，影响关联条款时一并调整并提示 |
| 3 | 生成修改版文档，提供下载链接 |
| 4 | 说明修改内容、理由及影响的关联条款 |
| 5 | 修改≥3处或跨多条款时，提供版本对比摘要 |

修订标记（Track Changes）：作者统一设为 `Pacgate Law Firm`，每轮以上一轮客户上传的确认版为基础。

## 工作流示例

- **中日跨境展览服务合同**：跨境服务委托，中国法，排他性 → 仲裁更优（纽约公约）→ 16章：管理费+实报实销双轨计费 → Playbook 有利版 → 中英双语 Word + 起草说明 + 合规检查清单
